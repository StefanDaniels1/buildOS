# Repository Cleanup Analysis

## Core System Architecture

The buildOS system has a simple architecture:

```
run_with_dashboard.py (wrapper)
  ↓
run.py (orchestrator)
  ↓
tools/ifc_analysis_tool.py (MCP tools)
  ↓
.claude/agents/batch-processor.md (agent definition)
```

## Files Analysis

### ✅ KEEP - Essential Core Files

#### Orchestration
- **run.py** - Main orchestrator (ESSENTIAL)
  - Uses Claude Agent SDK
  - Spawns batch-processor agents
  - Instrumented with observability

- **run_with_dashboard.py** - Observability wrapper (ESSENTIAL)
  - Wraps run.py
  - Sends dashboard events
  - Environment variable setup

- **run_buildos.sh** - Automated runner (KEEP)

  - Auto-starts dashboard
  - Convenient wrapper

- **tools/ifc_analysis_tool.py** - MCP tool definitions (ESSENTIAL)

- **pyproject.toml** - Python dependencies (ESSENTIAL)

- **uv.lock** - Locked dependencies (ESSENTIAL)
